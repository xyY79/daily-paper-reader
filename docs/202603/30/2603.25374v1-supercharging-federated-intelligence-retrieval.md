## 摘要
检索增强生成（RAG）通常假设对文档进行集中式访问，但当知识分布在私有数据孤岛中时，这种假设就不再成立。我们提出了一种使用 Flower 构建的安全联邦 RAG 系统，该系统执行本地孤岛检索，而服务器端聚合和文本生成则在经过认证的机密计算环境中运行，即使在存在“诚实但好奇”或被攻破的服务器的情况下，也能实现机密的远程大语言模型（LLM）推理。我们还提出了一种级联推理方法，在不削弱机密性的情况下，将非机密第三方模型（例如 Amazon Nova）作为辅助上下文引入。

## Abstract
RAG typically assumes centralized access to documents, which breaks down when knowledge is distributed across private data silos. We propose a secure Federated RAG system built using Flower that performs local silo retrieval, while server-side aggregation and text generation run inside an attested, confidential compute environment, enabling confidential remote LLM inference even in the presence of honest-but-curious or compromised servers. We also propose a cascading inference approach that incorporates a non-confidential third-party model (e.g., Amazon Nova) as auxiliary context without weakening confidentiality.

### 1. 核心问题与研究动机
*   **核心问题**：传统的检索增强生成（RAG）假设可以集中访问所有文档，但在现实中，医疗、金融等敏感领域的数据往往分布在不同的私有孤岛中，且受法规限制无法汇总。
*   **研究动机**：现有的联邦 RAG（FedRAG）虽然解决了数据本地化问题，但检索到的文档在传输和聚合过程中通常以明文形式暴露给服务器，存在隐私泄露风险。
*   **目标**：构建一个端到端的安全联邦 RAG 系统，确保从本地检索、服务端聚合到最终 LLM 推理的全过程机密性。

### 2. 方法论
*   **核心思想**：结合 **Flower 联邦学习框架**与**机密计算（Confidential Computing）**技术，在受信任执行环境（TEE）中处理敏感的聚合与推理任务。
*   **关键技术细节**：
    *   **本地检索**：每个数据孤岛（客户端）维护本地 FAISS 索引，根据查询检索 top-k 相关文档。
    *   **机密聚合**：服务器运行在 TEE 内，通过远程度量（Attestation）确保环境安全。服务器接收各孤岛返回的文档和评分，使用**倒数排名融合（Reciprocal Rank Fusion, RRF）**算法进行重排序。
    *   **三种推理模式**：
        1.  **独立推理（Standalone）**：在服务器 TEE 内的 CPU 上运行轻量级 LLM（如 SmolLM）。
        2.  **级联推理（Cascading）**：在 TEE 内结合本地 LLM 与外部非机密第三方模型（如 AWS Nova）。第三方模型提供辅助概率知识，最终由 TEE 内模型整合生成，在不泄露原始文档的前提下提升性能。
        3.  **机密推理（Confidential）**：利用 **Flower CRC** 服务，在远程加固的 GPU TEE 中运行大规模 LLM（如 Qwen3），实现高性能且隐私受控的推理。

### 3. 实验设计
*   **数据集/场景**：使用了 MedRAG Toolkit 中的四个医疗语料库：PubMed（23.9M 片段）、StatPearls、Textbooks 和 Wikipedia。
*   **Benchmark**：MIRAGE 基准测试，包含三个任务：PubMedQA（是/否/可能）、BioASQ（是/否）和 MedQA（多选题）。
*   **对比方法**：对比了上述三种推理模式（Standalone vs. Cascading vs. Confidential），评估指标为准确率（Accuracy）和端到端查询时间（Time）。

*   **服务端（TEE）**：采用纯 CPU 环境运行 SmolLM 1.7B 模型，未雇佣 GPU。

*   **第三方模型**：调用了 Amazon Nova Micro 接口。
*   **客户端**：4 个联邦客户端，分别存储不同的医疗数据集。

*   **实验规模**：针对 3 个主流医疗问答数据集进行了完整测试，涵盖了从判断题到多选题的不同难度。

*   **公平性**：所有模式均在相同的联邦客户端和检索参数（k=8, kRRF=60）下进行，具有较好的客观性。

### 6. 主要结论与发现
*   **级联推理的有效性**：即使使用轻量级的外部模型（Nova Micro），级联模式在 PubMedQA 和 MedQA 上的准确率分别提升了约 40% 和 46%，证明了“混合信任”架构的实用价值。
*   **机密推理的优越性**：Flower CRC（Qwen3 235B）在所有任务中准确率最高，且由于 GPU 加速，其延迟（约 25s）远低于 CPU 上的轻量级模型（约 43-48s）。
*   **安全性验证**：系统成功实现了在服务器不可信（诚实但好奇）的情况下，保护检索文档不被泄露给基础设施提供商。

*   **端到端隐私保护**：不仅保护了原始数据，还通过 TEE 保护了中间聚合结果和推理 Prompt。
*   **灵活的推理架构**：提出的级联推理允许在受限的 TEE 算力下，利用强大的外部 API 增强生成效果，而无需担心核心隐私。
*   **工程落地性强**：基于成熟的 Flower 框架开发，为分布式私有数据的智能检索提供了可行的工业化方案。
