Abstract 

This paper addresses the challenge of improving information retrieval from semi-structured eXtensible Markup Language (XML) documents. Traditional information retrieval systems (IRS) often overlook user-specific needs and return identical results for the same query, despite di ff erences in users’ knowledge, preferences, and objec-tives. We integrate external semantic resources, namely a domain ontology and user profiles, into the retrieval process. Documents, queries, and user profiles are represented as vectors of weighted concepts. The ontology applies a concept-weighting mechanism that emphasizes highly specific concepts, as lower-level nodes in the hierarchy provide more precise and targeted information. Relevance is assessed using semantic similarity mea-sures that capture conceptual relationships beyond keyword matching, enabling personalized and fine-grained matching among user profiles, queries, and documents. Experimental results show that combining ontologies with user profiles improves retrieval e ff ectiveness, achieving higher precision and recall than keyword-based approaches. Overall, the proposed framework enhances the relevance and adaptability of XML search results, supporting more user-centered retrieval. 

Keywords: Information retrieval, User profile, Domain ontology, Personalized search, XML, Vector space model 

1. Introduction 

Information Retrieval (IR) is a fundamental field of computer science, aiming to identify and extract doc-uments from large collections based on their relevance to a user’s information needs, typically expressed as textual queries [1]. An Information Retrieval System (IRS) generally encompasses three core tasks: represent-ing document content, modeling the user’s query or information need, and comparing these representations to compute a relevance score. One of the enduring challenges in IR lies in accurately modeling relevance so as to deliver the most meaningful and useful results [2]. The rapid digitization of content and advances in web technologies have led to exponential growth in avail-able data. Over time, document formats have evolved from flat, unstructured text to semi-structured represen-tations such as SGML and, more prominently, XML—a widely adopted standard for data representation. XML is particularly well suited for IR because it combines structural richness with descriptive content, enabling fine-grained and personalized access to information [3]. Despite notable progress in indexing and querying techniques, personalized search systems still face important limitations. Many approaches rely on user pro-file models that are either overly generic or insu ffi ciently expressive, failing to capture the complexity of user preferences [4]. Moreover, the semantic integration between user preferences and document structure is often shallow, neglecting the hierarchical granularity of XML documents and the richness of conceptual relationships [5]. Most existing work also focuses primarily on the profiling phase without providing mechanisms for the continuous adaptation of profiles as user queries evolve [6]. To address these gaps, we propose an approach that integrates semantic user profile modeling via a domain ontology with structural exploitation of XML documents. Personalization is achieved by constructing a user profile that reflects individual interests and incorporating it into the search process to improve result relevance. Our system represents user interests as weighted ontology-based vectors, enabling fine semantic interpretation of queries. Profiles are dynamically updated after each user interaction through a progressive reinforcement mechanism for the most relevant concepts.

The choice of XML as the core document representation is motivated by its structured nature and semantic richness, which are key features for fine-grained, personalized IR. Its hierarchical organization supports precise content modeling, while its compatibility with ontology languages such as OWL facilitates semantic integration [5, 7]. XML also o ff ers a flexible framework for representing both data and user preferences, making it a strong candidate for adaptive and intelligent search systems. While other formats such as JSON, YAML, Markdown, or NoSQL data models are prevalent in modern development (particularly for lightweight structures and e ffi cient data exchange), they have limitations in rep-resenting rich hierarchies and semantic annotations. JSON, for instance, is e ffi cient for API-based data transfer but lacks native typing, complex schema validation, and smooth integration with Semantic Web standards such as RDF or OWL [8]. YAML and Markdown are even more minimalist, making them unsuitable for deep seman-tic structuring [9]. NoSQL databases, including document-based (MongoDB) and column-based (Cassandra) systems, excel in scalability but do not provide standardized frameworks for fine-grained semantics, semantic validation, or automated reasoning [10]. XML thus remains a strategic choice, combining explicit structure, W3C standardization, compatibility with semantic technologies, and flexible processing through XPath /XQuery. These features make it an ideal medium for personalized IR in complex, information-rich environments. Recent studies highlight XML’s potential in personalized search, leveraging profile extraction, semantic transformation, and structured indexing techniques [11]. However, few o ff er a coherent integration of ontology, XML structure, and dynamic online profile adapta-tion. Our work addresses this gap by providing a unified framework that combines ontological representation, dynamic vectorization of user interests, and semantic exploitation of XML documents to improve the accuracy, personalization, and relevance of retrieved results. The main contributions of this paper are summarized as follows: - We propose a personalized information retrieval framework for semi-structured XML documents that integrates a domain ontology with user profiles to improve result relevance. - We represent documents, queries, and user profiles as ontology-based weighted concept vectors, and introduce a concept-weighting mechanism that emphasizes more specific (lower-level) ontology concepts. - We design a personalized matching and ranking process based on semantic similarity, coupled with a dynamic user-profile updating mechanism that progressively reinforces concepts inferred from user inter-actions. - We validate the proposed approach through experiments on an XML document collection, showing im-proved retrieval e ff ectiveness (precision, recall, and F1-score) compared to a non-weighted baseline, and we will make the source code and experimental data publicly available to support reproducibility. The remainder of this paper is organized as follows: Section 2 reviews related work on XML indexing, semantic retrieval, and personalization. Section 3 introduces the domain ontology, the concept-weighting mech-anism, and the XML document representation model. Section 4 presents the proposed personalized indexing and retrieval approach. Section 5 reports the experimental setup and results. Section 6 concludes the paper and outlines future directions.

3Ontologies and semantic resources. Another stream integrates semantic knowledge to go beyond keyword matching. Domain ontologies provide explicit concepts and relations that support semantic indexing and query expansion, illustrated by Paz-Trillo et al. [17], Ibrahim et al. [18], and Martin et al. [19]. Ontology-driven IR has also been explored in several application domains, including e-learning [20], academic research [21], journalism [22], and image retrieval [23]. In addition to domain ontologies, general lexical resources such as WordNet have been used for semantic indexing [24, 25], often in combination with traditional keyword-based representations. 

Personalized information retrieval. A further stream addresses personalization through user profiles, which capture contextual signals such as interests, preferences, goals, and interaction history [47]. Profiles can be acquired explicitly [26] or inferred implicitly from behavior [27], and represented using a variety of models including vectors, semantic /hierarchical structures, probabilistic models, graphs, and neural representations [28–35]. Integration typically occurs either during query reformulation [4, 36–41] or during matching and ranking [29, 33, 42]. Because user interests may evolve, adaptive updating of profiles is commonly highlighted as an important requirement [48]. 

Personalized IR for XML documents. Finally, several studies combine user modeling with XML retrieval, show-ing that profiles can improve e ff ectiveness in semi-structured search [19, 43, 44]. Existing approaches include XML-based preference /profile representations with keyword-oriented ranking [45, 46] and integrations of IR techniques with native XML databases for e ffi cient structural access [11]. As reflected in Figure 1, prior research often emphasizes either (i) XML structure-aware retrieval without deep semantic user modeling, or (ii) personalization approaches that do not fully exploit ontology-based concept weighting and the hierarchical granularity of XML. In this paper, we bridge these directions by integrating a domain ontology and a dynamically updated user profile, representing documents, queries, and profiles as weighted concept vectors to enable fine-grained semantic matching for personalized XML retrieval. 

3. Preliminaries 

3.1. Domain ontology 

Building on prior work in semantic indexing, we propose an approach to indexing XML documents based on a domain ontology. This ontology extends the one used in [49] and is inspired by the OBIREX (Ontology-Based Information Retrieval for XML documents) ontology developed in [50]. OBIREX was originally designed for e-learning in computer science and structures pedagogical knowledge through hierarchical and interrelated concepts, supporting semantic annotation of resources and improving the relevance of search results. In our study, the adopted knowledge division provides a generic, modular, and reusable framework for organizing domain concepts around the following entities: 

• Domain : a set of alternatives from which the user selects. 

• Path : a learning or navigation path that specifies a vocabulary supporting domain understanding. 

• Field : an area in which a study activity is carried out. 

• Element : a teaching unit that can be combined with others to personalize search. 

• Script : a set of knowledge checks characterized by prerequisite elements. 

• Concept : a set of evaluated granules that may be linked to other concepts by various relationships. 

• Granule : a knowledge unit within a script, presented alone or combined with other granules. Figure 2 illustrates the resulting domain ontology, including invariant elements shared across domains. The developed ontology includes invariant elements that are shared across domains. For example, the com-puter science domain ontology is obtained by instantiating the generic ontology described above and organizing computer science knowledge into the classes domain , path , field , element , script , concept , and granule . Ontology concept weighting. In our approach, we aim to improve the relevance of information retrieval (IR) by exploiting our domain ontology. When both a parent concept and one of its child concepts can describe an 5element, we select the child concept because it is more specific and therefore more informative. To emphasize specific concepts (i.e., those located deeper in the ontology hierarchy), we assign weights to ontology concepts. We assume that the total weight of the ontology equals one (1), i.e., it is the sum of the weights of all its concepts. The weight is distributed across ontology levels so that concepts at depth i receive higher weights than those at depth i − 1. Coe ffi cients are assigned as follows: 

• Each parent concept receives a coe ffi cient equal to its depth in the ontology. 

• If a concept has several parent concepts, its coe ffi cient is the average of its parents’ coe ffi cients (sum divided by the number of parents). 

• Coe ffi cients are propagated toward leaf concepts, starting from the highest depth, and increased only when moving from depth i to a deeper depth i + 1. This assignment favors descendant concepts, especially leaf concepts. We denote the resulting coe ffi cient by the function Coef( Ck). Figure 4a illustrates an example of coe ffi cient assignment. • successors lie above-right. Using intervals, structural relationships between two nodes u and v are resolved as follows: 

• ancestor–descendant : u is an ancestor of v iff the interval of u contains the interval of v;

• predecessor–successor : u precedes v iff the interval of u precedes the interval of v.Figure 8 presents the result of applying this technique to the XML document represented in Figure 5. To extract node descriptors, we parse XML documents using a parser. Two common parser families are: 

• event-oriented parsers: API SAX (Simple API for XML); 

• tree-oriented parsers: API DOM (Document Object Model). We use SAX callbacks to compute descriptors, mainly: 

• startDocument : invoked when a new document is encountered; 

• startElement : invoked when an opening tag is encountered; 

• endElement : invoked when a closing tag is encountered; 

• characters : invoked when text is encountered. A stack is used to store start values of nodes whose end value is not yet known: 

• on startElement : push the start value onto the stack; 

• on endElement : read the start value from the stack top, pop it, and obtain the parent from the new stack top. These steps generate descriptors of the form ( element _name , start , end , parent ), which support navigation, subtree inclusion, and filtering during indexing and retrieval. Figure 9 shows the evolution of the stack as the XML document is traversed. The algorithm can also compute additional attributes such as node height (stack depth minus one) and path (stack content). For implementation, we store descriptors in a relational database using the following tables: 

• Document (Idf_doc, doc_name )

• Element (idf_doc, begin, ele_name, end, parent )

• Attribute (idf_doc, begin, end, parent, att_name, value_att )

• Text (idf_doc, begin, value, end, parent )

4.4. Implementation 

The proposed approach is implemented as a web application. We chose the Java programming language and Servlets for their flexibility and portability. This work is situated in the Semantic Web context, which aims to enrich content with machine-interpretable meaning. Accordingly, we use OWL (Web Ontology Language) to represent the domain ontology and the Jena API for ontology creation, manipulation, and access. Figure 10 presents the overall architecture of our prototype. As shown in Figure 10, after registration, the user provides personal information and an initial profile is created, where the centers of interest are assigned uniform default weights. The user then submits information needs in the form of queries; these queries are stored in the profile database and used to continuously enrich the profile by updating the centers of interest according to the user’s searches. 5.1. Settings 

To validate our approach, we evaluate it on a collection of XML documents from the computer science domain, using the domain ontology introduced in Section 3.1. The experiments target personalized information retrieval in semi-structured XML documents. Although reference collections such as INEX [61], TREC [62], and CLEF [63] are widely used, they do not fully meet our needs. In particular, while INEX is XML-oriented, it is di ffi cult to exploit for reliably mapping documents to the concepts defined in our ontology. Moreover, INEX o ffi cially ceased in 2015, which limits its suitability for recent personalized settings [64]. More generally, these collections do not explicitly address personalization or the dynamic evolution of user preferences in semi-structured environments. For these reasons, we built our own test database. It contains 400 XML documents covering several com-puter science topics, organized according to the concepts of our domain ontology. Conclusion 

This paper presented a personalized XML information retrieval approach that combines a domain ontology with a dynamically updated user interest vector. By weighting ontology concepts and continuously adapting profile weights based on user queries, the system exploits explicit semantic structure while accommodating evolving preferences. Experimental results, including F1-scores across di ff erent user profiles, indicate that the approach improves retrieval e ff ectiveness, particularly for specific concepts supported by the ontology hierarchy. Future work will focus on validating the approach on larger and more heterogeneous XML collections and on extending it beyond the computer science domain. We also plan to investigate neural methods to refine user interest vectors from both explicit and implicit interaction signals, and to explore automatic ontology enrichment from XML corpora. Additional directions include modeling collective user profiles for collaborative personalization and integrating a proactive recommendation module to anticipate information needs.
