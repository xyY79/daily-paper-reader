## 摘要
LLM 重排序受限于通过向量相似度检索到的前 k 个文档，这既无法实现上下文查询-文档 Token 交互，也无法捕捉多模态相关性分布。虽然 LLM 查询重构尝试通过生成改进的或额外的查询来提高召回率，但其后仍需进行向量相似度检索。因此，我们提出通过引入 ReBOL 来解决这些前 k 个检索阶段的失效问题。ReBOL：1) 利用 LLM 查询重构来初始化关于文档相关性的多模态贝叶斯优化（BO）后验分布；2) 迭代地获取文档批次进行 LLM 查询-文档相关性评分，随后更新后验分布以优化相关性。在探索了查询重构和文档批次多样化技术后，我们在五个 BEIR 数据集上，使用两种 LLM（Gemini-2.5-Flash-Lite, GPT-5.2）将 ReBOL 与 LLM 重排序基线进行了评估。ReBOL 始终能获得更高的召回率和具有竞争力的排名，例如在 Robust04 数据集上，与最佳 LLM 重排序器相比，其 recall@100 为 46.5% 对 35.0%，NDCG@10 为 63.6% 对 61.2%。我们还展示了 ReBOL 可以实现与 LLM 重排序器相当的延迟。

## Abstract
LLM-reranking is limited by the top-k documents retrieved by vector similarity, which neither enables contextual query-document token interactions nor captures multimodal relevance distributions. While LLM query reformulation attempts to improve recall by generating improved or additional queries, it is still followed by vector similarity retrieval. We thus propose to address these top-k retrieval stage failures by introducing ReBOL, which 1) uses LLM query reformulations to initialize a multimodal Bayesian Optimization (BO) posterior over document relevance, and 2) iteratively acquires document batches for LLM query-document relevance scoring followed by posterior updates to optimize relevance. After exploring query reformulation and document batch diversification techniques, we evaluate ReBOL against LLM reranker baselines on five BEIR datasets and using two LLMs (Gemini-2.5-Flash-Lite, GPT-5.2). ReBOL consistently achieves higher recall and competitive rankings, for example compared to the best LLM reranker on the Robust04 dataset with 46.5% vs. 35.0% recall@100 and 63.6% vs. 61.2% NDCG@10. We also show that ReBOL can achieve comparable latency to LLM rerankers.

论文针对当前“检索-重排序”（Retrieve-then-Rerank）流水线的核心瓶颈提出了挑战。传统方法依赖向量相似度进行初步检索，但向量检索存在两个主要缺陷：
*   **表达能力受限**：仅依靠向量点积，无法捕捉查询与文档之间复杂的 Token 级交互。
*   **分布假设单一**：向量检索通常诱导单模态的相关性分布（即相关文档都簇拥在查询向量周围），而实际相关性往往分布在多个不同的语义簇中（多模态）。
即使使用 LLM 进行查询重构，最终仍需回归向量检索。**ReBOL** 的核心含义是将检索过程重新建模为一个**贝叶斯优化（BO）**问题，通过 LLM 的深度语义理解能力动态引导搜索过程，从而突破传统 top-k 检索的召回率上限。

#### 2. 方法论
ReBOL（**Re**trieval via **B**ayesian **O**ptimization with **L**LM）的核心思想是利用高斯过程（GP）在文档嵌入空间上建立相关性后验分布，并迭代地选择最有价值的文档进行 LLM 评分。

*   **初始化（多模态先验）**：利用原始查询 $q_0$ 和 LLM 生成的多个重构查询 $\{q_1, ..., q_Q\}$ 作为初始观测点。假设这些点具有最高相关性（如 3 分），从而在嵌入空间中初始化出多个相关性峰值，形成多模态后验分布。
*   **迭代观测循环**：
    1.  **采集（Acquisition）**：使用采集函数（如 Greedy 贪婪搜索或 UCB 上置信界）从全量文档库中选择一个批次（Batch）的文档。
    2.  **批次多样化**：引入了基于 **MMR（最大边界相关性）** 的改进采集函数，确保选中的文档批次在空间上具有代表性，避免陷入局部最优。
    3.  **LLM 评分**：调用 LLM 对选定文档进行绝对相关性评分（0-3 分，Pointwise 模式）。
    4.  **后验更新**：将 LLM 的评分作为新的观测值更新高斯过程，修正相关性曲面。
*   **最终排序**：在迭代结束后，根据高斯过程的后验均值对文档进行最终排名。
